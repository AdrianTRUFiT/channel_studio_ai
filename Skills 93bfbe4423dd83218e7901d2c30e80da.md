# Skills

In your architecture, Skills = Capabilities that agents can reliably perform, independent of the pipeline they’re embedded in.

Think of skills as atomic, reusable, testable functions that agents can call to complete their mission.

Below is the full breakdown — the Skill Taxonomy, the Skill Contracts, and the Skill Wiring inside your sovereign AI stack.

---

⭐ The Skill Taxonomy (Your System’s Capability Map)

Every agent in your system draws from five categories of skills:

1. Retrieval Skills

How an agent pulls information from memory.

- SQL queries
• Vector similarity search
• File/manual retrieval
• ICP lookup
• Pattern lookup

These skills give the agent context.

---

1. Interpretation Skills

How an agent understands the retrieved information.

- Pain extraction
• Intent detection
• Pattern matching
• Signal classification
• Sentiment analysis
• Role inference

These skills give the agent meaning.

---

1. Reasoning Skills

How an agent makes decisions.

- Lead scoring
• Lead qualification
• Routing decisions
• Offer matching
• Fulfillment logic
• Upsell timing

These skills give the agent judgment.

---

1. Generation Skills

How an agent produces structured output.

- Delivery messages
• Nurture sequences
• Summaries
• ICP profiles
• Opportunity maps
• Content drafts

These skills give the agent expression.

---

1. Action Skills

How an agent interacts with the system.

- Write to DB
• Write to logs
• Update lead status
• Trigger next pipeline
• Save artifacts
• Send notifications

These skills give the agent agency.

---

🧩 How Skills Fit Into the Agent Architecture

Every agent has:

- A role
• A mission

Skills are the verbs of the agent.

Here’s how they wire in:

Pipeline → Agent
Agent → Recall Skills
Agent → Interpretation Skills
Agent → Reasoning Skills
Agent → Generation Skills
Agent → Action Skills
Pipeline → Next Stage

This is the intelligence loop.

🧱 Skill Contracts (How Skills Are Defined)

Every skill has a Skill Contract:

Skill Name:
Purpose:
Inputs:
Outputs:
Memory Used:
LLM Required:
Constraints:

Example:

Skill Name: ExtractPainSignals
Purpose: Identify pains, frustrations, unmet needs
Inputs: post_body
Outputs: pains[], frustrations[], unmet_needs[], severity
Memory Used: none
LLM Required: yes
Constraints: JSON only, severity 1–10

This makes skills testable, reusable, and composable.

---

🔌 How Skills Wire Into Agents

Agents call skills like functions:

context = self.recall(lead)
pain = self.skills.extract_pain(context)
score = self.skills.score_lead(context)
route = self.skills.route_lead(context)

Skills are not pipelines.
Skills are not agents.
Skills are capabilities.

Agents are compositions of skills.

Pipelines are orchestrations of agents.
